Test Driven Development (TDD)

Was ist TDD?

Wie funktioniert TDD?

  • Schreiben von Tests vor der Implementierung
  • Entwicklungsprozess
  • Unabhängige Testfälle

Vorteile

  • Frühe Fehlererkennung
  • Lebendige Dokumentation
  • Rückverfolgbarkeit
  • Vermeidung von Over-Engineering

Nachteile

  • Zeitaufwand
  • Lernkurve
  • Anpassung an bestehenden Code
  • Zu viele Tests

Continous Integration (CI)

Was ist CI?

Kontinuierliche Integration
von Code in ein gemeinsames Repository

Welche Ziele verfolgt CI?

  • Automatisierung
  • Regelmäßige Integration
  • Auto-Tests
  • Frühzeitige Fehlererkennung
  • Einfachere Zusammenarbeit im Team
  • Bessere Verwaltung und Transparenz
  • Schnellere Markteinführungen

CI in 6 easy steps:

  1. Write and push your code
  2. Build automatically
  3. Test automatically
  4. Get your feedback
  5. Integrate into the main repository
  6. Continuous Integration
